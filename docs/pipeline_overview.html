O que o pipeline faz (em linguagem simples)

Pega os dados brutos do nosso banco de dados interno (Data Lake), calcula todas as métricas que antes eram feitas por fórmulas no Google Sheets (base inicial, novos, churn, upgrades, pagamentos, etc.), e disponibiliza o resultado pronto pra consulta — sem depender de planilha pesada.

Fluxo do Pipeline

1. Data Lake (Trino)
Banco de dados interno da OLX com todos os dados de receita, contratos e pagamentos.
hive.planejamento.re_gold_receita_unificado_air, hive.planejamento.re_silver_planos_periodicos_cb, hive.planejamento.re_silver_receita_cb_paids_air
↓ Cloud VPC
2. Extração (Python)
Script conecta ao banco, baixa os dados do dia e salva em formato otimizado (Parquet).
pipeline_local.py → 5 tabelas extraídas (~840k registros principal)
↓
3. Google Cloud Storage (GCS)
Armazém temporário na nuvem. Os arquivos ficam aqui antes de ir pro BigQuery.
gs://teste-extracao-trino/
↓
4. BigQuery (Tabelas Brutas)
Dados brutos carregados no armazém analítico do Google. Mesma estrutura do banco de origem.
Ex: conect-python-g-sheets.planejamento_comercial.re_gold_receita_unificado_air
↓ Automático
5. Transformações SQL (Tabelas Derivadas)
9 queries SQL calculam as métricas de receita: base inicial, novos, churn, upgrades, downgrades, pagamentos, migrações, diarização. É aqui que a "inteligência" das fórmulas do Sheets é replicada.
sql/receita_enriquecida.sql → sql/bd_full.sql → sql/diarizacao.sql (etc.)
↓
6. BigQuery (Tabelas Finais)
Resultado pronto pra consumo. Cada tabela replica uma aba do Sheets antigo.
bd_full, diarizacao, cb_pagamentos, bd_planos_uf, bd_planos_mensais_sva, etc.
↓
7. Google Sheets (Connected Sheets)
Planilha "Receita 5.0" conectada diretamente ao BigQuery. Atualiza com um clique, sem fórmulas pesadas.
Receita 5.0 → BigQuery Connector → tabelas finais
